# PROJECT_RULES

## 1. ワークフローとツール運用
- **テスト実行の必須化**
  - 開発サイクルおよび「Next Steps」において、以下3点の実行を義務付ける。
    - `npm run lint`: 静的解析
    - `npm run test`: ユニット/コンポーネントテスト
    - `npm run test:e2e`: E2Eシナリオテスト
- **デバッグ指針**
  - テスト失敗時は、単に再試行せず、以下を確認・記録する手順を徹底する。
    - どの assertion が落ちたか
    - コンポーネントのレンダリング状態 (debug出力)
    - APIレスポンスの状態

## 2. コーディング規約
- **Modal Focus Management (重要)**
  - モーダル実装時は、`useEffect` の依存配列に `onClose` を含めると再レンダリングループの原因になる可能性がある。
  - `onClose` を `useRef` で保持するか、依存配列から意図的に外して安全に呼び出すパターンの使用を標準とする。
- **Type Safety & Sanitation**
  - `any` 型の使用を原則禁止し、適切な型定義を行う。
  - 入力値のバリデーションとサニタイズを徹底する。
- **Linter Strictness**
  - `no-undef`: error (未定義変数の厳密検知)

## 3. UI/UX 仕様 (必須実装要件)
- **共通ヘッダー**
  - ロゴの横にメインコンテンツ（例：「イベント一覧」）へのリンクを常設。
  - 右端のユーザーメニューはドロップダウン形式。
  - ドロップダウン内に「プロフィール編集」「サインアウト」を配置。
- **認証ユーザー向けUI**
  - ログインユーザー名とアイコンをヘッダー右上に、ロゴと高さを揃えて配置。
  - プロフィール編集画面を実装（ユーザー名、パスワード、アイコン、メール変更）。
  - パスワード入力欄には「表示/非表示」切り替え機能を必ず実装。
- **管理者向けUI**
  - ユーザー管理、利用履歴照会機能へのリンクを実装。

## 4. プロジェクト固有 UIデザイン指針
- **テーマ**: ナチュラル（アースカラー・柔らかい角）
- **ターゲット**: 学生、若手社会人、保護者
- **特徴**:
  - アイコンにはテキストラベルを併記。
  - 専門用語を避け、平易な言葉を使用。
  - 操作ミスを防ぐ確認ダイアログの実装。
  - フォーカス管理によるアクセシビリティ担保。

## 5. 技術スタック固定
- **Frontend**: React 18 + TypeScript + Vite 5
- **Style**: Tailwind CSS 3 (utility-first) + CSS Variables (tokens)
- **Auth/Backend**: Supabase (Auth, DB, RLS)
- **Testing**: Vitest (Unit), Playwright (E2E)
